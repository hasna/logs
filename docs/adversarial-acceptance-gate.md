# Adversarial Acceptance Gate

Last updated: 2026-06-17

Open-logs cannot be called a robust universal telemetry data substrate until adversarial reviewers agree that the evidence supports that claim.

## Gate Rule

Two independent adversarial reviews must pass after implementation and real-life validation.

The reviewers should be asked to reject the claim unless the implementation proves:

- Broad data capture across apps, processes, repos, machines, browsers, servers, builds, tests, agents, CLIs, and streams.
- Raw event payloads are durable in append-only filesystem segments.
- SQLite indexes are complete enough to locate, correlate, and reconstruct raw events.
- Producer event IDs and event times are preserved.
- Duplicate ingestion is idempotent where event IDs exist.
- Streams support resume, backpressure, and explicit overflow behavior.
- Recovery handles partial writes, crashes, corrupt lines, and index rebuild.
- Redaction occurs before raw persistence.
- Sensitive data does not sync by default.
- Derived projections can be rebuilt from raw evidence.
- Real-life validation scenarios pass under isolated data dirs.

## Automatic Fail Conditions

Any of these fail the gate:

- Raw payloads only exist in SQLite.
- Events cannot be reconstructed by segment path, byte offset, and length.
- The stream can silently skip live events.
- Ingest accepts arbitrary unvalidated payloads on network endpoints.
- Secrets can be found unredacted in raw segments after redaction tests.
- Generated local data appears as untracked repo files by default.
- A crash can create silent event loss without recovery reporting.
- Sync uploads secrets, auth credentials, prompts, env values, or replay payloads by default.

## Required Evidence Bundle

The final review bundle should include:

- Verification command list and results.
- Segment verifier output.
- SQLite integrity check.
- Sample raw segment record with sensitive values redacted.
- Query results proving segment pointer reconstruction.
- Real-time stream reconnect evidence.
- Crash recovery evidence.
- Redaction grep evidence using canary secrets.
- Larger multi-producer/high-volume evidence beyond the accepted bounded 5,000-event run, including million-event, crash-during-stress, fsync-on durability, and longer soak results.
- Multi-machine or simulated sync evidence.
- Dogfood summary for open-logs itself.
- Remaining risks and accepted limitations.
